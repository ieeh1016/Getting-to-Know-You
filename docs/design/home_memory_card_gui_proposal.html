서로의 기억은 런처가 아니라
최근 기억 한 장처럼 보여야 한다

현재 홈의 `서로의 기억`은 기능 설명, 공간 카운트, 최신 preview, 두 개의 CTA가 모두 같은 힘으로 보여서 초점이 흐립니다. 홈에서는 “최근에 남긴 기억”을 먼저 보여주고, 공간 수와 만들기는 조용한 보조 정보로 낮추는 방향을 제안합니다.

현재 구조

아이콘과 설명, 두 사람의 공간 카운트, 최신 기억 preview, `기억 보기`와 `카드 만들기` 버튼이 한 카드 안에 순서대로 쌓입니다.

애매한 느낌

최근 기억이 핵심인지, 두 공간 통계가 핵심인지, 새 카드를 만들라는 기능 안내인지 한눈에 결정되지 않습니다.

유지할 규칙

홈 주요 카드와 메뉴 진입은 유지합니다. 하단 탭을 추가하지 않고, private 카드는 상대 카운트와 preview에 드러나지 않습니다.

바꿀 방향

카드 전체를 “최근 기억 열기”로 읽히게 만들고, 생성은 우측 작은 plus action, 공간 수는 하단 작은 요약으로 낮춥니다.

현재 상태

기능 런처와
기억 preview가 섞임

두 개의 카운트 tile과 두 개의 버튼이 강해서, 홈 카드가 “최근 기억”보다 “기능 소개 박스”처럼 느껴집니다.

결론과 구현 메모

추천 결론

홈 카드의 최상위 목적을 `최근 기억 1장 열어보기`로 정합니다. `카드 만들기`는 작은 plus action으로 낮추고, 공간 카운트는 카드 하단의 작은 상태 bar로 이동합니다.

Firestore write 경계

홈에서 카드 preview를 보거나 기억 화면을 여는 행동은 Firestore write를 만들지 않습니다. 새 카드 저장, 수정 요청 저장, 수정 반영 같은 explicit action만 기존 문서 write를 만듭니다.

확인할 테스트

카드 없음, 최신 shared 카드 있음, private 카드만 있음, unread badge 있음, 긴 제목/본문 상태에서 390px 화면의 text와 bottom nav가 겹치지 않는지 확인합니다.
