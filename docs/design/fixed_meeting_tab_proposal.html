확정된 만나는 날만 모아 계획하기

`약속`은 날짜를 조율하는 곳으로 남기고, 새 `만남` 탭은 이미 정해진 날짜만 보여줍니다. 여기서 그날 뭐 할지, 어디 갈지, 어떤 순서로 움직일지를 차분하게 정합니다.

ROLE SPLIT

약속은 조율, 만남은 실행

캘린더에서 날짜를 정하고 나면, 그 이후의 결정은 새 탭으로 넘어갑니다. 한 화면에 너무 많은 책임을 주지 않습니다.

ONLY FIXED DAYS

확정된 날짜만 노출

후보 날짜, 조율 중인 날짜는 보이지 않습니다. 들어오자마자 “우리가 실제로 만나는 날”만 확인합니다.

SOFT PLANNING

딱딱한 일정표 대신 가벼운 코스

완료 체크리스트보다 `할 것`, `갈 곳`, `먹을 것` 후보를 정리하고 둘이 마음을 표시하는 정도로 둡니다.

SCREEN FLOW

새 `만남` 탭의 4개 주요 화면

하단 탭은 `홈 / 질문 / 음악 / 약속 / 만남 / 장소 / 마이`가 됩니다. 탭이 많아지는 대신 `약속`과 `만남`의 역할을 명확히 나눠 화면을 덜 복잡하게 만듭니다.

확정된 날이 없을 때

`만남` 탭은 후보를 만들지 않습니다. 확정된 날이 없으면 약속 탭으로 부드럽게 보내는 빈 상태만 보여줍니다.

데이터 구조

`ScheduleEntry.isMeetingDay == true`인 날짜만 `만남` 탭에 노출합니다. 플랜은 1차로 `meetingPlans` 또는 `meetingPlanItems`에 저장하고, 장소는 `placeId` 참조를 선택적으로 둡니다.

탭 수 문제

하단 탭이 7개가 되므로 글자 크기와 간격을 더 조밀하게 잡아야 합니다. 부담이 크면 `장소`를 `만남` 안으로 일부 흡수하는 2차 안도 가능합니다.

추천 구현 순서

먼저 `만남` 탭 목록과 상세를 만들고, 그다음 장소 후보 가져오기를 붙입니다. 지도 검색 자체는 기존 `장소` 기능을 재사용합니다.
