구현 방향 메모

MVP 범위
  • 대화 회고와 자동 생성은 제외하고, 사용자가 직접 만드는 기억 카드만 제공합니다.
  • 하단 네비게이션에는 추가하지 않고, 홈 주요 카드와 기능 모아보기 메뉴에서 진입합니다.
  • 기억 화면 안에서는 영우의 공간, 민영이의 공간처럼 작성자 기준 공간을 나누어 카드를 모읍니다.
  • 카드 유형은 좋아하는 것, 싫어하는 것, 요즘 이야기, 함께 할 것, 조심할 것 정도로 시작합니다.
공유와 안전장치
  • 카드 공개 범위는 `서로 보기`와 `나만 보기`를 카드마다 선택합니다.
  • 상대는 공유 카드에 `맞아`, `좋아`, `조금 수정`으로 반응할 수 있습니다.
  • 수정 요청은 원문을 즉시 바꾸지 않고, 작성자가 `반영하기`를 눌러야 적용됩니다.
Firestore write 경계
  • 타이핑, 탭 이동, 필터 변경, 카드 열람은 Firestore write를 만들지 않습니다.
  • 카드 저장, 공개 범위 변경, 반응 등록, 수정 요청, 요청 반영에서만 Firestore write가 발생합니다.
  • 나만 보기 카드는 작성자 ownerUid만 읽고 쓸 수 있어야 합니다.
데이터 모델과 테스트
  • `memoryCards`: ownerUid, subjectUid, type, title, body, visibility, createdAt, updatedAt.
  • `memoryCardResponses`: cardId, responderUid, reaction, correctionText, status, createdAt.
  • 390px 폭에서 홈 주요 카드, 메뉴 행, 공간 탭, 공개 범위 버튼, 기존 하단 탭 텍스트가 겹치지 않는지 확인합니다.
