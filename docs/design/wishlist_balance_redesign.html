구현으로 이어질 때의 제안 범위

두 기능 모두 Firestore 구조를 크게 바꾸지 않고 UI/UX를 먼저 개선할 수 있습니다. 위시는 기존 wish 문서에 lane/status 계산을 얹고, 밸런스는 기존 selection 문서를 기반으로 히스토리와 요약을 파생합니다.
