TASTE MATCH · RESULT PRIVACY

취향 매치는
고르는 화면과 결과 화면을 분리해야 해요

선택 직후 결과 카드만 숨겨도, 아래 기록에서 같음/다름이나 상대 선택이 보이면 이미 결과를 본 것과 같습니다. 그래서 이번 개편은 “메인 화면은 고르는 곳”, “내 취향 노트는 내 기록만”, “결과함은 내가 열어본 결과만”으로 정보 경계를 다시 나눕니다.

RULE 01

선택 화면에는 결과성 정보를 두지 않기

선택 직후에는 내 선택 저장 상태와 이유 입력만 보여줍니다. 상대 선택 여부, 같음/다름, 비교 문장은 노출하지 않습니다.

RULE 02

기록은 “내 취향 노트”로 재정의

기록 화면은 내가 고른 선택과 내가 쓴 이유를 다시 보는 곳입니다. 결과 기록과 섞지 않습니다.

RULE 03

결과는 명시적으로 열어야 공개

상대 선택이 있어도 결과는 자동 공개하지 않습니다. 사용자가 `결과 보기`를 눌렀을 때만 비교 정보를 보여줍니다.

RULE 04

결과함은 잠금 상태를 먼저 보여주기

아직 안 본 결과는 잠금 카드로 보여줍니다. 열어본 결과만 같음/다름과 연결 액션을 표시합니다.

SCREEN PROPOSAL

화면 흐름 제안

기존 한 화면 안에 있던 선택, 기록, 결과를 네 개 상태로 분리합니다. 사용자가 심리적으로 “내가 고른다”와 “서로 비교한다”를 다른 행동으로 느끼게 하는 것이 핵심입니다.

DEVELOPMENT BRIEF

개발 기안

핵심은 화면 UI보다 데이터 공개 상태입니다. 선택 데이터와 결과 공개 상태를 분리해서, 사용자가 결과를 열기 전까지 어떤 화면에서도 상대 선택/같음/다름이 새지 않게 만듭니다.

정보 구조

기존 `취향 기록`은 결과가 섞이기 쉬우므로 이름과 역할을 분리합니다.

1
취향 매치오늘의 카드 선택, 내 선택 저장, 이유 입력만 담당합니다.
2
내 취향 노트내 선택과 내 이유만 다시 보여줍니다. 상대 선택과 결과 상태는 숨깁니다.
3
결과함열 수 있음/기다림/열어본 결과를 분리합니다. 열기 전 카드는 스포일러 없이 잠금 상태입니다.
4
결과 상세사용자가 명시적으로 연 결과만 같음/다름, 상대 선택, 연결 액션을 보여줍니다.

추천 데이터 모델

결과 공개는 개인별 행동이므로 각 사용자의 선택 문서에 공개 여부를 저장하는 방식이 가장 단순합니다.

balanceSelections 기존 필드 `questionId`, `profileId`, `optionId`, `reason` 유지. 여기에 `resultRevealedAt` optional timestamp를 추가합니다.
공개 가능 조건 내 선택과 상대 선택이 모두 존재할 때만 `결과 열어보기` 버튼을 활성화합니다. 단, 버튼 전에는 상대 optionId를 UI에 렌더링하지 않습니다.
공개 완료 조건 내 selection 문서의 `resultRevealedAt != null`일 때만 결과함의 `열어본 결과`에 비교 내용을 표시합니다.
Rules 영향 `balanceSelections`에 `resultRevealedAt` optional timestamp/null 허용을 추가하면 됩니다. 본인 `profileId == request.auth.uid` 조건은 그대로 유지합니다.

구현 단계

  1. 현재 `BalanceRecordSection`을 메인 화면에서 제거하거나 `내 취향 노트` 컴포넌트로 이름/역할을 변경합니다.
  2. `내 취향 노트`에서는 `controller.balanceSelectionFor(question)`과 `balanceReasonFor(question)`만 표시합니다.
  3. `partnerBalanceSelectionFor(question)`은 `결과 보기` 또는 `resultRevealedAt` 조건을 만족할 때만 UI에 전달합니다.
  4. `결과함` 섹션을 추가해 `열 수 있음`, `기다림`, `열어본 결과` 필터를 제공합니다.
  5. `결과 열어보기`를 누르면 `resultRevealedAt`을 저장하고 결과 상세를 표시합니다.

검증 기준

  • 선택 직후 화면 어디에도 상대 선택, 같음/다름, 결과 문장이 보이지 않는다.
  • 내 취향 노트에는 내 선택과 내 이유만 보인다.
  • 결과함의 잠금 카드는 결과가 준비됐다는 사실만 보여주고 비교 내용을 숨긴다.
  • `결과 열어보기`를 누른 뒤에만 상대 선택과 비교 문장이 보인다.
  • 한 번 열어본 결과는 새로고침 후에도 `열어본 결과`로 유지된다.
  • Firebase mode에서 실제 상대 선택이 없으면 결과 열기 버튼이 비활성 또는 미노출된다.

중요한 UX 판단

결과 공개를 너무 숨기면 기능의 재미가 약해질 수 있습니다. 그래서 “결과가 준비됐어요”는 알려주되, “무슨 결과인지”는 숨기는 방식이 가장 균형이 좋습니다. 사용자는 호기심을 느끼고, 동시에 원치 않는 순간에 결과를 강제로 보지 않습니다.
