구현 범위 제안

1차는 `ScheduleEntry.meetingPlanItems`에 문자열 리스트로 저장합니다. 할 것/갈 곳 타입은 UI label로만 가볍게 두고, 복잡한 체크리스트 완료 상태는 넣지 않습니다.

장소 기능과의 관계

장소 보드에서 날짜 연결을 다시 노출하지 않습니다. 약속 화면에서만 `장소에서 가져오기`를 제공해 목적이 분명할 때만 플랜에 담습니다.

저장 타이밍

입력 중에는 Firestore write를 만들지 않고, `그날 정리 저장` 또는 `만나는 날 저장`을 눌렀을 때만 저장합니다. 무료 플랜 경계와 기존 저장 UX를 유지합니다.
