이전안이 별로였던 지점

이번 방향

댓글은 답변과 분리하되 독립 리스트가 되지는 않습니다. 답변 카드 아래에 작은 꼬리 달린 shelf처럼 붙여 “답에 딸린 한마디”로 보이게 합니다.

유지할 기능

한 답변에 내 댓글 하나, 받은 댓글 하나라는 현재 모델을 유지합니다. 작성 전에는 한 줄 composer, 저장 후에는 작은 댓글 bubble로 바뀝니다.

결론과 구현 메모

추천 결론

댓글을 thread/list로 키우는 안은 버리고, 답변 카드 하단에 작은 shelf로 붙이는 안을 추천합니다. 홈에서 댓글 관계는 보이되 기능 설명 냄새가 줄어듭니다.

Firestore write 경계

draft typing은 계속 local state only입니다. `저장`, `수정 저장`, `다시 시도` 같은 explicit action만 기존 comment document create/update Firestore write를 만듭니다.

확인할 테스트

받은 댓글 있음, 내 댓글 없음, 작성 중, 저장됨, 오류 상태에서 390px 화면의 text와 bottom nav가 겹치지 않는지 확인합니다.
