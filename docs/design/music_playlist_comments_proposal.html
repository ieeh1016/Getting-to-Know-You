구현 방향 초안

  • 댓글은 MusicNote 자체 필드보다 별도 MusicNoteComment 모델로 둡니다.
  • 필드는 id, musicNoteId, body, createdByProfileId, createdLabel, createdAt, updatedAt 정도가 적당합니다.
  • 댓글 작성, 수정, 삭제만 Firestore write를 만들고 입력 중인 draft는 로컬 상태에만 둡니다.
  • 목록 정렬은 기존 음악 노트의 updatedAt 기준을 유지합니다. 댓글이 달렸다고 곡 카드가 위로 튀지 않게 합니다.
  • 목록에서는 note별 댓글 수와 최신 댓글 preview만 계산해 보여줍니다.
  • 파트너가 남긴 새 댓글은 홈 요약 또는 음악 탭의 작은 badge로만 알립니다.
